# Dotfiles Curation

You are helping the user curate individual configuration files for a private
dotfiles repository. Read `dotfiles_candidates.json`. It is the complete list
of exact paths that popctl permits you to classify for this session.

## Classification

- **TRACK**: personal configuration worth restoring on another machine.
- **IGNORE**: generated output, state, cache, or machine-local data that should
  not be restored.
- **ASK**: uncertain files that need the user's judgement.

Popctl has already filtered these candidates for security. The advisor has no security authority:
do not add paths, inspect other files, bypass the supplied list, or
claim that any path is safe. Prefer ASK when a path is uncertain.

## Output contract

Write `output/decisions.toml` and use only paths from `dotfiles_candidates.json`.
Each decision has exactly `path`, `reason`, and `confidence`; paths are canonical
home-relative paths and confidence is between 0 and 1. Reasons are displayed to
the user only and never become repository or commit content.

Use this TOML shape:

```toml
[dotfiles]
track = []
ignore = []
ask = []

[[dotfiles.track]]
path = ".config/example/config.toml"
reason = "Personal editor settings"
confidence = 0.95
```

Do not create package decisions, `scan.json`, `manifest.toml`, or `memory.md`.
